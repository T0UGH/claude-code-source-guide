## 01｜一次请求怎么进入 Claude Code 的主循环

### 主问题
一条用户请求进入 Claude Code 之后，整条动态主线大致会如何展开，为什么卷二必须先从“主循环总图”开始？

### 核心判断句
**Claude Code 的主循环不是“收输入、回输出”的单步过程，而是一条把请求接入运行时、形成当前判断、触发能力、接回结果并持续推进的工作链。**

### 这篇必须完成的任务
- 作为卷二总起篇，把整条主线先亮出来
- 让读者知道卷二后面几篇分别在拆哪一步
- 把卷一第 3 篇的低分辨率动态闭环，提升成卷二的正式总图
- 建立卷二与卷三、卷四的边界感

### 这篇不讲什么
- 不深挖 QueryEngine 源码细节
- 不展开输入预处理细节
- 不进入工具执行层具体实现
- 不把这一篇写成组件清单

### Mermaid 主图
1. 卷二主循环总流程图

## 05｜系统怎么决定这一轮要不要调用能力

### 主问题
在主循环里，模型什么时候只是回答，什么时候会产出“该做什么”的当前决策，并把这一轮推进到执行层？

### 核心判断句
**在 Claude Code 里，模型的关键产出不只是答案，而是“下一步该不该做事、该做什么”的当前决策。**

### 这篇必须完成的任务

- 不深挖 tool execution 内部实现

## 07｜一轮 Agent Turn 什么时候继续，什么时候收口

### 主问题
一轮 agent turn 的边界到底由什么决定，为什么不是“输出一句话就算结束”，而是看当前结果是否足够收口？

### 核心判断句
