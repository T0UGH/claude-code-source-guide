## 七个认知台阶分别回答什么

### 卷一：Claude Code 是什么系统
回答：**Claude Code 到底是什么系统，它的核心对象和基本轮廓是什么。**

### 卷二：一次 Agent Turn 是怎么跑起来的
回答：**一条用户请求怎样真正进入系统，并闭合成一轮 agent turn。**

### 卷三：工具系统怎么把模型意图落成执行
回答：**模型已经决定做事之后，runtime 怎样把意图真正落成现实动作。**

### 卷四：上下文与状态怎么维持系统持续工作
回答：**Claude Code 为什么不是跑一轮就散，而是能持续工作、能恢复、能长期维持。**

### 卷五：外部扩展与多代理能力
回答：**这套系统怎样长出更多能力，并把系统之外的能力源接进来，最终收成平台层。**

### 卷六：多 agent 协作运行时
回答：**Claude Code 为什么不只是“多开几个 agent”，而是已经长出一层正式协作运行时。**

### 卷七：命令、工作流与产品层整合
回答：**用户入口、运行时接口、工作流控制层，最后怎样被收成今天这个产品形态。**

## 这版重组和旧结构的关键区别

这次重组最关键的变化，不是换了几个标题，而是把整套内容改成了明确的认知递进：

- 卷优先于文章
- 主问题优先于材料归档
- 认知递进优先于源码目录
- 卷与卷之间必须形成自然导流
- 每一卷只解决自己这一层问题，不提前抢下一卷的职责

也就是说：

> **读者不是在逛源码目录，而是在一层一层理解 Claude Code 这套系统。**
