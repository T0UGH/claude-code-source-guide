## 01｜总阅读诊断
主判断已清楚，但原先偏“卷四为什么存在”的说明味较重；现已压缩为“持续工作系统 vs 短事务系统”的直接对照。

- 删除了部分元解释段落，避免作者解释自己的写法。
- 保留一张总图，帮助读者先留下“工作线要被维持”的框架。
- 当前残留：读者应能记住“卷四不是长上下文机制卷，而是持续工作卷”。

## 02｜总阅读诊断
原风险是术语并排、像对象字典；现已改成“每层对象分别回答什么问题”。

- 强化了 session / transcript / messages / system prompt / context 的职责分工。
- 去掉了若干低收益重复定义。
- 当前残留：读者应能记住“context 不是大桶，session 也不是更大的 context”。

## 04｜总阅读诊断
原风险是写成 token 窗口常识文；现已收回到“为什么持续工作一定逼出治理”。

- 把重心从“长度变长”改成“密度失衡 + 当前重点被埋”。
- 保留一张失效链图，帮助读者把治理必要性和持续工作问题接起来。
- 当前残留：读者应能记住“治理不是为了省 token，而是为了保住工作能力”。

## 05｜总阅读诊断
原风险是后半卷机制目录；现已强调这是链，不是四个 feature 并列。

- 明确 06 / 07 / 08 的职责拆分，减少后文串线风险。
- 保留“视图治理 -> 主动减负 -> 恢复续接”的主链。
- 当前残留：读者应能记住“后半卷在讲一条链，而不是四个功能名词”。

## 06｜总阅读诊断
最容易被读成抽象概念文；现已把问题压到一句：治理对象先是工作视图，不是 transcript。

- 删掉了若干会让读者感觉“你在重新换一种说法重复同一判断”的段落。
- 保留 transcript / 工作视图 对照图，帮助读者建立分层感。
- 当前残留：读者应能记住“当前 query 看到的东西，不等于历史档案本体”。

## 08｜总阅读诊断
原风险是恢复机制说明书；现已加重卷尾收束职责，把前文重新压成持续工作总图。

- 强化 recovery / restore 的两步分工。
- 强化卷尾收束，避免文章只剩“恢复怎么做”。
- 当前残留：读者应能记住“卷四最终讲的是工作线怎样重新接活”。

## 总体结论

这一版卷四关键篇章的读者风险，主要已经从：

- 机制目录感
- 术语桶感
- 元解释过多
- 逻辑正确但残留不稳

收敛为：

- 前半卷在立持续工作骨架
- 中段在立治理必要性与治理链总图
- 后半卷在回答“治理的是什么、怎样减负、怎样续接”

仍需最终横向检查的是：07 不要抢 06 的“治理对象校正”，08 不要只剩恢复说明而丢掉卷尾总收束。
