# 卷一｜Claude Code 的运行时底座

## 这一卷在讲什么

这一卷不打算先去回答“Claude Code 到底是什么”这种大问题。那样很容易写成空泛总览，也容易把后面几卷的东西提前讲乱。

这一卷只做一件事：

> 把 Claude Code 的运行时底座先搭清楚。

更具体一点，就是先把下面四层看明白：

- **tool**：Claude Code 真正能调用的执行原语是什么
- **agent**：这些原语是怎么被装配成一个可运行体的
- **skill**：能力单元是怎么被定义、发现、接入和执行的
- **prompt**：这些能力最后怎么被整理成模型能理解的说明层

所以这一卷不是“学习路线”，也不是“全书入口”，而是一卷结构卷。读完它之后，你应该已经对 Claude Code 的底层骨架有一个稳的印象。到了第二卷再去看 QueryEngine 主链，就不会一上来被名词砸晕。

---

## 为什么要这样排

第一卷原先的问题，不是内容不够，而是东西太杂。

早期总图、学习路线、探索期概览、主循环前置、skill 相关的不同层次，全都挤在前面。单篇看没问题，放在一卷里就会打架。读者会不断遇到这种情况：刚在看工具，下一篇突然跳到主循环；刚在看 skill 接口，下一篇又像在讲全书导航。

所以这次重排做了一个很直接的决定：

> 第一卷只保留真正服务“运行时底座”的内容。

这也是为什么这卷最后按下面的顺序来：

> **tool → agent → skill → prompt**

这个顺序不是为了排得好看，而是因为它最符合这套系统在读者脑子里该怎么长出来。先看执行原语，再看运行体怎么装起来，再看能力单元怎么接进去，最后再看这一切怎么被翻译成模型可用的说明。

---

## 这一卷适合谁读

这一卷比较适合两类人：

### 1. 想系统读 Claude Code 源码的人
如果你不是只想查一个点，而是想建立整体认识，这一卷应该先读。

### 2. 已经在用 coding agent，但总觉得内部结构有点模糊的人
你可能知道 Claude Code 能调工具、会分 agent、支持 skill，也大概知道 prompt 很重要，但这些东西怎么接起来还不够清楚。这一卷就是把这几个层面重新梳顺。

这一卷不太适合纯新手扫盲。它不是“十分钟认识 Claude Code”，而是“把底座先看明白”。门槛不算低，但也不是写给少数内核开发者看的。只要愿意顺着读下去，大多数工程师都能跟上。

---

## 读这一卷前，不需要先懂什么

你不需要预先搞懂：

- QueryEngine 的完整调用链
- 长上下文压缩机制
- MCP / hooks / plugin 的外部扩展体系
- 权限系统和 team runtime 的全部细节

这些都不是这一卷的任务。

你只需要接受一个前提：

> Claude Code 不是“会聊天的 CLI”，而是一套 agent runtime。

一旦把这个前提放稳，这一卷后面的结构就会顺很多。

---

## 建议阅读顺序

### Part 1：Tool
先把 Claude Code 能实际动用的执行原语看清楚。

1. **05 Tool 总图与 BashTool 入口**  
   从工具层切入整卷，是这卷最合适的正文入口。
2. **07 Tool 体系总览**  
   看工具不是怎么一个个堆出来的，而是它们在系统里怎么分层。
3. **10 BashTool 不是跑个 shell 这么简单**  
   BashTool 是理解整个工具层最重要的例子之一。
4. **11 FileReadTool 统一读取入口**  
   理解读取原语。
5. **12 FileEditTool 是受控修改原语**  
   理解受控编辑原语。
6. **13 FileWriteTool 是整文件覆盖原语**  
   理解整文件写入原语。
7. **14 GrepTool 是结构化内容搜索原语**  
   理解搜索原语。
8. **17 ToolSearchTool 是工具发现层**  
   看工具不只是被调用，还要能被发现。
9. **08 主循环里 tool 怎么被接住**  
   作为 tool 段收束，解释工具最终怎么接进 runtime。

### Part 2：Agent
工具层看完之后，再看 Claude Code 怎么把这些东西装成可运行的 agent。

10. **15 AgentTool 不是再开一个 agent 这么简单**  
    进入 agent 层最自然的一篇。
11. **19 runAgent 是把 agent 组装成可运行体的装配线**  
    这一段的核心文。
12. **20 forkSubagent 是在同一上下文里分叉出 worker**  
    理解 agent runtime 的分叉能力。
13. **21 loadAgentsDir 是 agent 定义层的总入口**  
    看 agent 定义从哪里进入系统。
14. **22 builtInAgents 是官方内建角色模板总表**  
    看模板层，帮助理解 agent 是怎么被组织的。

### Part 3：Skill
这一段会比前两段更长，因为 skill 在 Claude Code 里确实很重要，而且它不是一个单点功能。

15. **16 SkillTool 是把 skill 接进 runtime 的桥**  
    从 skill 接入层起步。
16. **24 loadSkillsDir 是 skill 定义层的总入口**  
    进入 skill 定义层。
17. **25 SkillTool 是 skill 进入执行层的总入口**  
    看 skill 怎么真正进入执行层。
18. **26 forkedAgent 是 skill 接入 agent 执行层的胶水层**  
    看 skill 和 agent 是怎么连起来的。
19. **27 runAgent 是 skill 接进 agent 执行层的主干**  
    看 skill 如何进入主干执行链。
20. **28 processPromptSlashCommand 是 skill-inline 路径的展开层**  
    把另一条真实入口也补上。
21. **02 Skill frontmatter 字段速查**  
    放到这里读更顺，因为这时你已经知道 skill 真在系统里干什么了。
22. **29 frontmatter 不是注释而是 skill 的运行时接口**  
    继续把接口意义补完整。
23. **30 什么样的 SKILL 才是真正能进 runtime 的好 skill**  
    从“怎么接”转到“什么样才成立”。
24. **31 skillify 是 Claude Code 对好 skill 写法的官方样板**  
    看官方样板。
25. **32 verify 是 Claude Code 对验证即运行时观察的官方定义**  
    看官方如何理解 verify。
26. **33 debug 是面向当前 session 的现场诊断 skill**  
    看 debug skill 的定位。
27. **34 skills 篇结语：skill 不是提示词而是能力单元**  
    skill 段收束。
28. **35 skill 的 name 和 description 是什么时候让 LLM 感知到的**  
    进入 skill 发现机制。
29. **36 when_to_use 和 description 哪个对 skill 发现更关键**  
    继续讲发现机制。
30. **37 skill 和 agent 是怎么分工的**  
    用一篇收清 skill 和 agent 的边界。

### Part 4：Prompt
前面三层都看完后，再用一篇 prompt 收尾。

31. **23 prompt 是 AgentTool 给模型看的使用说明层**  
    这一篇适合放在最后读。它不是孤立的一层，而是把前面那些能力重新组织给模型看。

## 这一卷删掉了什么，为什么删

这次重排删掉了几篇早期文章：

- `00 学习路线`
- `01 Skill 总图`
- `03 Skill / Fork / Agent / Review`
- `04 环境变量有效性校验`
- `06 Claude Code tools 总览`

删掉不是因为它们当时写得不好，而是因为它们现在已经不再服务这一卷的结构目标。

有些属于早期导航文，今天应该由 guidebook 首页和阅读路径承担；有些属于重叠总览，留下来只会让第一卷重新发散；还有些过细，放在主手册里不划算。

这卷还有两篇从第一卷挪到了第二卷：

- `09 从 query 到 tool-call 的连接处`
- `18 主循环在编排一次完整 agent 回合`

原因也很简单：

> 它们已经明显进入主循环视角，应该留给 QueryEngine 那一卷，不该继续挤在底座卷里。

## 读完这一卷，你应该得到什么

如果这一卷整理得对，读完以后你至少应该拿到下面这些东西：

### 1. 你知道 Claude Code 的底座不是一团糊
它至少可以稳定分成 tool、agent、skill、prompt 这几层。

### 2. 你知道 skill 为什么值得单独讲那么长
因为 skill 不是“提示词模板”，而是 Claude Code 这套 runtime 里很关键的一层能力单元。

### 3. 你知道第二卷要接什么
第二卷不再负责补前置，而是正式讲一条请求是怎么跑完整个系统的。

### 4. 你脑子里已经有一个比较稳的系统图
不一定每个细节都记住了，但至少不会再把工具层、agent 层、skill 层和 prompt 层混在一起。
