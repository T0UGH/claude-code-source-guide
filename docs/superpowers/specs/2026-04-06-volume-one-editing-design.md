# 卷一正文整理设计（中整理版）

## 目标

把卷一从“已有文章集合”整理成一组真正能连续阅读的手册正文。

这次采用 **中整理版**，不是只搬运，也不是大规模重写，而是在现有文章基础上做一轮足够明显、但不过火的整理。

## 整理范围

卷一按既定结构处理：

1. Tool 段
2. Agent 段
3. Skill 段
4. Prompt 收尾

处理对象以映射表 v2 为准。

## 要做的事情

### 1. 迁入卷一正文
把卷一相关旧文从知识库迁入新仓库，放到 `docs/guidebook/volume-1/` 下，按卷内顺序命名。

### 2. 标题统一
标题从“研究笔记标题”改成“手册正文标题”，但不追求过度包装。

原则：
- 更清楚
- 更适合公开阅读
- 不故作花哨

### 3. 补导语
每篇补一段简短导语，说明：
- 这篇在卷一里的位置
- 它解决什么问题
- 建议怎么接着读

### 4. 加导航
每篇至少补：
- 所属卷
- 卷内位置
- 上一篇
- 下一篇
- 必要时的前置篇目

### 5. 做弱去重
只处理明显重复的开场、重复解释和职责打架的地方。

允许：
- 删除重复开场
- 把重复解释缩成一句
- 调整局部段落顺序
- 明确两篇文章的边界

不做：
- 大规模重写整篇正文
- 改核心判断
- 合并文章
- 重新发明一套完全不同的结构

## 分段执行方式

为了避免一次改太多，卷一按四段依次推进：

1. Tool
2. Agent
3. Skill
4. Prompt

每段完成后都应形成一个能读的局部块，而不是先摊一地半成品。

## 文件结构建议

```text
docs/guidebook/
  README.md
  00-mapping-table-v2.md
  01-volume-one-runtime-foundation.md
  volume-1/
    01-tool-overview-and-bash-entry.md
    02-tool-system-overview.md
    ...
```

## 成功标准

卷一整理完成后，应满足：

1. 卷一正文已全部迁入新仓库
2. 每篇标题适合公开阅读
3. 每篇都有简短导语和导航
4. 明显重复开场已经处理
5. 按 tool → agent → skill → prompt 读下来是顺的

## 当前决定

- 采用中整理版
- 不做大重写
- 分段执行
- 先把卷一做成能连续阅读的一卷，再考虑卷二及后续卷
